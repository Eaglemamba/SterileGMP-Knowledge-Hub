Annex C — Designing compliant labels using a building block approach

積木式標籤設計：從 SSCC 基礎到 DSCSA 序列化擴展 | p37–p43

C.1–C.2 標籤定義與積木概念

Label（標籤）指套用在包裝上的印刷區域，包含文字與／或條碼資料（線性條碼、複合式或 2D 符號）。可分開印製並在不同階段黏貼，組合成完整標籤。

積木式結構（building block）是標籤格式的基本標準單元，每個積木或子積木可包含：

  • 文字或圖形
  • 條碼符號（2D 或線性，含人可讀詮釋）
  • 空白區

積木可個別製作或與其他積木結合，支援「資料產生後即印製」的彈性流程。積木應以垂直堆疊方式排列。

C.3–C.4 LPB 文字高度 & 區段

文字高度以 Lines-Per-Block (LPB) 定義，共 9 種（1–8 及 10 LPB），由標籤印製商依設備能力選定實際字高。標準定義三個資訊區段（segment），由上而下堆疊：

  1. Carrier segment（承運人區段）— 置頂
  2. Consignee segment（收貨人區段）
  3. Shipper segment（發貨人區段）— 置底

C.5–C.6 標籤尺寸與格式規範

標籤尺寸不固定，由標籤印製商依貿易夥伴需求、設備能力及運輸包裝大小決定。

積木尺寸規範（Table C.1 摘要）

每個積木最多可分為 4 個子積木（sub-block），以垂直分隔線區分。文字積木／子積木不得包含條碼符號；條碼應置於獨立的條碼符號積木中。

* 字高含 ascenders、descenders 及 leading。字高寬比不超過 2:1（以「M」字元量測）。

積木式設計與 Pharma CDMO 應用解析

比喻：積木式標籤 = 樂高積木

想像一張出貨標籤是由不同顏色的樂高積木垂直疊起：最頂層是給快遞掃描用的 SSCC 積木，中間是給客戶用的 GTIN + Lot + Expiry 積木，最底層是發貨人資訊積木。每層可獨立印製、替換，不影響其他層。

這對 CDMO 特別實用：序列號（serial number）通常在最後包裝工序才確定，用積木設計就能在前段先貼 GTIN/Lot 積木，最後再貼上序列號積木，完成 DSCSA 合規標籤。

LPB 選用原則 — CDMO 實務

  • 標題文字（Title）：固定 6 LPB，大寫，左對齊，最多 2 行
  • 主要資料（如 Lot No.、Exp Date）：建議 3–4 LPB
  • 次要資料（如 PO#、重量）：5–6 LPB
  • 條碼 HRI（人可讀詮釋）：通常 5–7 LPB，置於條碼下方

同一標籤上各 LPB 尺寸需有明顯視覺差異；8 LPB 必須明顯小於 7 LPB。

Pharma 積木建構順序：SSCC → GTIN → Serial

Step 1 — 製造 / 包裝階段
GTIN (AI 01) + Lot (AI 10) + Exp (AI 17) → 條碼積木 #1
↓
Step 2 — 出貨準備
+ SSCC (AI 00) → Carrier 積木 (置頂)
↓
Step 3 — DSCSA / EU FMD 序列化
+ Serial No. (AI 21) + NCC (AI 422) → 序列積木貼覆

重點：多階段貼標的合規風險

ISO 15394 允許「分段貼標」，但 GMP 要求每次更動均需書面紀錄。CDMO 必須在 BPR（批次記錄）中明確記錄各積木印製時間點、操作人員及核驗結果，以符合 21 CFR Part 211.68 及 EU GMP Annex 15 要求。

D.1 總則

ISO 15394 是框架標準，各產業的具體應用指引（application guideline）應在遵循本標準的前提下進一步細化。Annex D 列出 12 項產業指引需定義的要素。

D.2–D.12 需定義的 12 項要素

D.10 跨產業標籤的 crossover 效應

當供應商的出貨標籤從本產業流向域外客戶（例如 pharma CDMO 出貨給非 pharma 的零售通路），或域外供應商的標籤流入本產業時，雙方均應以 ISO 15394 本文規定的品質等級為基準。產業指引可設定更高要求，但不可低於本標準。

Pharma vs. Automotive AIAG — 應用指引差異比較

比喻：應用指引 = 建築法規地方版

ISO 15394 = 國際建築基準法（最低標準）。各產業的 application guideline = 地方建築法規（在基準法之上加嚴）。Pharma 的 GS1 Healthcare 標籤規範就像「醫療建築特殊規定」，比一般建築要求更嚴格的防火、衛生標準。

CDMO 實務：選擇 AI vs. DI 的判斷原則

  • 出貨給 pharma distributor / hospital：使用 GS1 AI（GS1-128 或 DataMatrix）
  • 出貨給 automotive / industrial 客戶：可能需要 ANSI MH10.8.2 DI + Code 39/128
  • 同一出貨箱供應多產業客戶：建議同時印製 GS1 + DI 條碼（見 Annex G）
  • 客戶合約優先：當客戶合約指定符號學，優先遵循合約；ISO 15394 為底線

Pharma 容器貼標重點解析

棧板雙面貼標原則（Pallet — 最重要）

ISO 15394 要求棧板至少貼兩張標籤（Figure F.1b）。GS1 Healthcare 進一步建議：

  • 四面均貼標（360° 可讀）：適用於自動倉儲（ASRS）或高速輸送帶掃描
  • 至少兩個相鄰面（90° 夾角）：確保叉車從任何角度靠近均可掃到
  • 標籤底部距地板：400–800 mm（避開叉車叉齒干擾及地面污染）
  • 頂部距棧板頂面：≤ 100 mm（便於高位掃描槍讀取）

小容器（Vial / Syringe / Ampule）貼標挑戰

ISO 15394 主要針對運輸單元（shipping unit）層級。對於 vial / syringe 等小型一次用容器，應參考 ISO 15223-1（醫療器材符號）、GS1 Healthcare Guideline for DataMatrix 及各市場 UDI 規範。

比喻：貼標位置 = 停車場出入口標誌

停車場的高度限制標誌必須在入口前夠遠的地方讓駕駛能看見，並且設在視線高度。條碼標籤的位置邏輯相同：放在掃描器可以「看到」的地方，不能被遮擋（shrink wrap / 標誌重疊），高度適合掃描器焦距。

CDMO 出貨貼標 SOP 重點

  • 紙箱：右下角主標籤 + 至少 1 面側貼；收縮膜包裝前確認條碼讀取率
  • 棧板：2 相鄰面各 1 張 SSCC 標籤；高度 400–600 mm
  • 冷鏈容器：標籤材質需耐 -80°C；考慮乾冰昇華時水氣冷凝對條碼的影響
  • IPC 過程中：在線掃碼驗證（in-process check），使用認證掃描器，拒收率 < 0.1%
G

Annex G — The impact of systems confronted with multiple symbologies and formats

多重符號學環境：自動辨別、移轉策略與設備升級 | p64–p67

G.1 概述 — 開放系統的多符號學問題

開放供應鏈中，掃描器可能遇到三種組合（見 5.1.1）：

  • Option (a)：GS1 Application Identifiers (AI) + GS1-128
  • Option (b)：ANSI MH10.8.2 Data Identifiers (DI) + Code 39
  • Option (c)：ANSI MH10.8.2 DI + Code 128

符號識別碼（Symbology Identifier，ISO/IEC 15424）是解碼器在資料前加入的前綴，不被編入條碼本身，由解碼器硬體自動附加輸出。

G.2 單一選項環境

僅掃描一種符號學時，主機系統應驗證符號識別碼，過濾掉不符合的資料流。Option (b) 用戶應關閉解碼器中其他符號學，防止誤讀。

G.3 多選項環境

支援 2–3 種選項的系統必須完整實作符號識別碼能力（Symbology Identifier capability）。組合「符號識別碼 + DI 或 AI」可提供可靠的資料輸入，自動辨別各選項。

G.4 移轉路徑（Migration choices）

可行的移轉路徑：

  1. Code 39 + DI → GS1-128 (AI)
  2. Code 39 + DI → Code 128 + DI
  3. Code 128 + DI → GS1-128 (AI)

所有移轉都需要相當長的並行期（parallel operation period），並需供應商、承運人、客戶三方協議。

G.4.3 設備需求

  • 印表機：需能正確產生新格式符號（含 FNC1 等控制字元）
  • 解碼器：設定為同時讀取新舊格式，並傳輸符號識別碼
  • 不支援符號識別碼的老舊解碼器應升級或汰換

G.5 移轉管理建議

G.5.1 產業主管機構（如 GS1 Taiwan、AIAG）應先調查設備現況、規劃升級路徑、協調三方，並排定舊格式廢止期。

G.5.2 標籤印製方應驗證映射軟體（AI/DI 格式差異，例如日期格式）、完整支援 FNC1，並在上線前進行印刷品質測試。

G.5.3 標籤掃描方應確保解碼器符合 ISO/IEC 15424，實作 AI 與 DI 格式驗證軟體，並完成資料格式轉換邏輯（AI 格式與 DI 格式的日期、單位差異）。

多重符號學 — CDMO 實戰解析

比喻：符號識別碼 = 語言標籤

想像條碼資料是電話語音留言，符號識別碼是系統自動在前面加上「這是中文留言」或「這是英文留言」的標記。主機系統（IT system）聽到「]C1」就知道是 GS1-128 的資料，用 AI 解析；聽到「]A0」就知道是 Code 39 + DI，用 DI 解析。沒有這個標記，系統就猜不出來該用哪套規則解讀。

GS1 + 非 GS1 同標籤的設計原則

當 CDMO 同時供應 pharma 客戶（需要 GS1 AI）與 automotive 客戶（需要 DI）時，可在同一標籤上印製兩種條碼：

  • 左側：GS1-128（含 GTIN + Lot + Exp，AI 格式）
  • 右側：Code 39（含 Part No. + Lot，DI 格式）
  • 主機系統透過符號識別碼自動辨別，分別路由到對應的業務邏輯
  • 兩者共享同一 Lot 資料，確保追溯一致性

CDMO 遷移計畫建議步驟

  1. 盤點現有掃描設備（是否支援符號識別碼輸出）
  2. 與客戶確認並行期（通常 6–12 個月）
  3. 更新 ERP/WMS 映射邏輯（AI ↔ DI 欄位對照）
  4. 印表機韌體升級，測試 FNC1 字元產生
  5. IQ/OQ/PQ 驗證新標籤系統
  6. 通知 QA 更新 SOP 及標籤規格書（Label Specification）

AI vs. DI 格式差異陷阱

移轉時，日期格式從 2 位年變為 4 位年是最常見的映射錯誤來源。

CDMO 出口多市場的標籤策略

  • 設計一標通用（universal label）：包含所有市場所需 AI（DataMatrix 含 01+10+17+21），印於單盒
  • 在 DataMatrix 之外另加 GS1-128 線性條碼，滿足仍需線性條碼的通路
  • NCC（National Code Container）或各市場特有欄位用 AI 90–99（內部約定）或獨立欄位處理
  • ERP 序列號管理需支援多市場：同一 GTIN 不同 serial format 的驗證規則

哪些市場不要求 unit-level 序列化（截至 2026）

  • 台灣（TFDA）：目前推廣但未強制 unit-level
  • 東南亞（泰、越、印尼）：多數仍在評估或試行
  • 部分非洲市場：只要求 batch-level 追溯
  • 澳洲（TGA）：目前無強制序列化要求
  • 印度（CDSCO）：已發布指引但執行時程多次延期

→ CDMO 應依最嚴格市場設計標籤，往下相容。

CDMO 出貨標籤合規清單

ISO 15394 + GS1 Healthcare + DSCSA / EU FMD 綜合查核清單

設計階段（Label Design Phase）

  • 已確認目標市場序列化要求（DSCSA / EU FMD / 其他）
  • 已選定正確符號學：GS1-128（unit carton + pallet）+ DataMatrix（unit of use）
  • 已包含所有必要 GS1 AI：01（GTIN）、10（Lot）、17（Exp）、21（Serial）
  • GTIN 已向 GS1 Taiwan（或對應組織）正確申請，格式為 14 碼
  • 標籤版面符合積木式設計（Carrier → Consignee → Shipper 由上而下）
  • 標籤尺寸符合容器尺寸，且各積木高度符合規範（25 mm ± 5 mm）
  • 冷鏈標籤已指定耐溫材質（Cryogenic-grade，最低 -80°C）
  • 標籤規格書（Label Specification）已記錄所有版面、字體、顏色、材質規格
  • Artwork 已通過 QA 審核，含 HRI（人可讀詮釋）核對

印製與驗證（Printing & Verification）

  • 印刷品質達 ISO/IEC 15416 / 15415 Grade ≥ 3.0（B）以上
  • 使用 660 nm 光源進行品質等級測定
  • DataMatrix x-dimension ≥ 0.375 mm（GS1 Healthcare 要求）
  • GS1-128 x-dimension 0.25–0.43 mm；quiet zone ≥ 10X
  • 每批印製前進行首件確認（First Article Inspection）
  • 條碼掃描驗證率（scan rate）≥ 99.9%，使用認證 ISO 驗證器
  • 序列號已從 ERP 正確傳輸至印表機，無重複序列號
  • 印製記錄（batch record）已留存，含 print timestamp + operator ID

貼標作業（Labelling Operation）

  • 貼標位置符合 Annex F（container type 對應正確位置）
  • 棧板標籤貼於兩個相鄰面，高度 400–800 mm
  • 標籤無皺折、氣泡、遮擋（shrink film 包裝後確認可讀）
  • Line clearance 已完成，確認無前批次標籤殘留
  • 標籤廢棄品（reject / spoilage）已計數並記錄於 BPR
  • 在線掃碼驗證（in-process scan）已執行，讀取率記錄於 BPR

電子追溯與上傳（Serialization / T3 / EMVS）

  • 序列號事件（commissioning）已上傳至對應平台（EMVS / VRS / NMPA 等）
  • T3（Transaction Data / Transaction Information / Transaction Statement）已備妥（DSCSA）
  • EPCIS 事件格式符合 GS1 EPCIS 1.2 / 2.0 標準
  • 出貨前已確認序列號狀態為「Active」（非 decommissioned）
  • 退貨 / 毀損流程已定義：序列號 decommission 流程 SOP 存在
  • MAH（市場授權持有人）已收到序列化數據，完成驗收

文件與變更管理（Documentation）

  • 標籤版本控制（label version control）已建立，變更需走 Change Control
  • 標籤供應商資格確認文件存在（print quality audit report）
  • 年度標籤審查（Annual Label Review）已排入 QMS 計畫
  • 符號識別碼（Symbology Identifier）設定已記錄於設備 IQ/OQ 文件
  • DSCSA / FMD 教育訓練已完成，含年度複訓記錄
  • 客戶稽核（customer audit）清單已包含標籤系統稽核項目

序列號格式建議（CDMO 自定義，AI 21）

格式建議：[SiteCode][Year][JulianDate][Sequence]

範例：
  TW2026097001234
  ├─ TW    = 台灣廠區代碼
  ├─ 2026  = 年度
  ├─ 097   = 第 097 天（儒略曆）
  └─ 001234= 當日流水號（6碼，補零）

最大長度：20 個英數字元（X..20）
禁用字元：# % / 空格（GS1 規範）
唯一性：需在品牌持有人的 GTIN + 有效期範圍內唯一

練習題 — ISO 15394 Annex C–G & GS1 應用

5 道情境練習題，含詳解

Q1 — 積木式標籤的分段貼標

您的 CDMO 接受委託製造一款注射劑，產品批次識別（GTIN + Lot + Exp）在包裝時即可確認，但序列號（Serial Number）需等到客戶 MAH 在歐洲 EMVS 系統申請後才能分配。請問依 ISO 15394 積木式設計，應如何規劃此標籤的分段印製與貼標流程？需說明哪些積木應在哪個階段印製，以及 GMP 文件要求。

點擊查看詳解

階段一（包裝時）：印製並貼附「產品積木」—— 包含 AI 01（GTIN）、AI 10（Lot No.）、AI 17（Expiry Date）的 DataMatrix 條碼積木 + 人可讀文字積木。此積木不應被後續積木覆蓋（Packaging label module）。

階段二（MAH 分配序列號後）：印製「序列積木」—— 包含 AI 21（Serial Number），製作為獨立的標籤積木，貼覆於預留空白區或以 overlay 方式貼附。

GMP 文件要求：BPR 中需記錄：(1) 各積木印製日期時間及操作員；(2) 序列號來源（由 MAH 通過哪個系統分配）；(3) 覆貼前的版面確認（包含照片記錄或電子驗證記錄）；(4) 廢棄標籤數量及銷毀記錄。符合 EU GMP Annex 16 及 FDA 21 CFR 211.68 要求。

Q2 — 多重符號學環境的解碼器設定

您的倉儲同時收到三種出貨標籤：(1) 藥品供應商使用 GS1-128 + AI；(2) 化工原料供應商使用 Code 39 + DI "J"；(3) 醫療設備供應商使用 Code 128 + DI。WMS 系統在掃描條碼時經常將 (a) 和 (c) 混淆，導致資料解析錯誤。請依 Annex G 的建議，說明應如何設定解碼器及 WMS 軟體來解決此問題。

點擊查看詳解

根源問題：GS1-128（option a）和 Code 128 + DI（option c）都使用 Code 128 物理符號，沒有符號識別碼就無法自動辨別。

解碼器設定：確認解碼器支援 ISO/IEC 15424 Symbology Identifiers 輸出。GS1-128 輸出 ]C1，Code 128 + DI 輸出 ]C0，Code 39 + DI 輸出 ]A0。若解碼器不支援此功能，需升級韌體或汰換設備。

WMS 軟體：在掃描資料前端讀取符號識別碼（2–3 字元前綴），依 ]C1 / ]C0 / ]A0 路由至不同解析邏輯：

  • ]C1 → GS1 AI 解析器（GTIN、Lot、Exp、Serial）
  • ]C0 → ANSI DI 解析器（Part No.、Lot、Qty）
  • ]A0 → DI "J" 解析器（Licence Plate）

Q3 — LPB 字高與條碼積木的設計衝突

標籤設計師在設計一張 100 mm × 150 mm 的藥品外箱標籤時，客戶要求在同一標籤上印製：GTIN、Lot No.、Expiry Date（文字）、DataMatrix 條碼、GS1-128 條碼（SSCC），以及 Ship-from / Ship-to 地址。設計師希望在同一個積木中同時放置文字和 DataMatrix，但你注意到這不符合規範。請指出問題所在，並提出符合 ISO 15394 Annex C 的正確版面配置。

點擊查看詳解

問題所在（Annex C.6.3）：文字積木（text building block）或文字子積木（text sub-block）不得包含條碼符號。DataMatrix 和文字必須分開置於不同積木。

正確版面配置（由上而下）：

  1. Carrier Segment（最上方）：雙高積木（51 mm）— GS1-128 SSCC 條碼 + HRI
  2. Consignee Segment：標準積木（25 mm）— Ship-to 地址文字，2 個子積木（名稱 + 地址）
  3. Shipper Segment：標準積木（25 mm）— Ship-from 地址文字
  4. Product Data 積木（條碼）：標準積木 — DataMatrix（含 GTIN + Lot + Exp + Serial）
  5. Product Data 積木（文字）：標準積木 — GTIN / Lot / Exp 人可讀文字，各為獨立子積木，6 LPB 標題 + 3 LPB 資料

Q4 — 冷鏈棧板標籤的材質與位置

您需要為一批 -80°C 超低溫凍乾原料藥（API）設計棧板運輸標籤，產品裝於乾冰冷凍容器（dry ice shipper）中，外層以透明塑膠膜包裹。標籤需在整個冷鏈運輸過程（-80°C 至到達現場 20°C 解凍前）保持可讀。請依 Annex F 及 GMP 實務，說明材質選擇、貼標位置及在線驗證要求。

點擊查看詳解

材質選擇：標籤紙需使用「Cryogenic-grade」合成紙或聚酯薄膜（Polyester / Polypropylene），不得使用普通感熱紙（在低溫下黏著劑失效）。黏著劑需選用耐低溫壓克力膠（Acrylic cold-resistant adhesive），測試溫度至少達 -80°C。條碼印刷建議熱轉印（Thermal Transfer）或雷射印刷，而非感熱印刷（DT）。

貼標位置（Annex F Figure F.1b）：棧板兩個相鄰面各貼一張標籤（4A + 4B 位置）。標籤位置：底部距地面 400–800 mm，避開乾冰昇華區（底部 100 mm 內）。透明塑膠膜包裹後，確認條碼在膜外仍可被直接掃描（不被水氣或冷凝遮蔽）。若外膜阻礙掃描，考慮在標籤外側加覆透明保護貼或使用標籤貼 pocket。

在線驗證：低溫環境掃描驗證應於包裝完成後、冷凍存放前執行，確認條碼讀取率 ≥ 99.9%（使用認證 ISO 驗證器）。另在解凍後（收貨方驗收時）再次掃描，確認冷鏈過程未損壞條碼。兩次掃描結果均記錄於 BPR。

Q5 — 全球多市場出貨的「一標通用」設計

貴公司 CDMO 客戶（MAH）計畫將同一批次藥品同時出口至美國（DSCSA）、歐盟（EU FMD）及中國（NMPA 追溯碼）。請說明在 ISO 15394 + GS1 框架下，如何設計一張能滿足三個市場要求的標籤，並指出哪些元素是通用的、哪些是市場特定的，以及如何在實務上處理「中國 20 位追溯碼」與 GS1 AI 並存的問題。

點擊查看詳解

通用元素（三市場共同）：

  • DataMatrix 含 AI 01（GTIN）+ AI 10（Lot）+ AI 17（Exp）+ AI 21（Serial）— 滿足 DSCSA 和 EU FMD 四要素，也是中國追溯碼的數據來源
  • GS1-128（SSCC）— 棧板 / 外箱追溯
  • 人可讀文字：Product Name、Lot No.、Exp Date（通用）

市場特定元素：

  • EU FMD：NCC（National Code Container）= 防偽封口完整性標記，需實體標籤
  • 中國：NMPA 20 位追溯碼，需另加一個 DataMatrix 條碼（或 QR Code），使用 AI 91 或 AI 240 編碼中國追溯碼，或將 NMPA 碼印於額外積木
  • 各市場語言 HRI：英文（美 / 歐）、中文（中國）

實務解決方案：在標籤上印製兩個 DataMatrix：(1) GS1 DataMatrix（含 4 個標準 AI）；(2) 中國 NMPA DataMatrix（AI 91 + 20 位碼）。使用 ISO 15394 的積木設計，將 NMPA 積木置於 GS1 積木旁側或下方，標注「CN Traceability Code」標題。電子追溯部分：GS1 EPCIS 上傳至 EMVS（EU）及 VRS（US）；NMPA 平台另行上傳中文格式資料。

Section 11: Inspection — 視覺檢驗標準

原文內容 Original Text

11.1 Visual Inspection

Any marking on the NIS essential for safe use shall remain visible, easily legible and indelible after pre-conditioning.

Inspection conditions:

  • Lighting: (215 ± 20) lux
  • Reading distance: 40 cm to 70 cm
  • Vision: normal or corrected-to-normal

Check for significant defects, including:

  • a) Markings no longer visible or legible (impacting safe functioning)
  • b) Cracks in body or components that might impact safe functioning
  • c) Compromised assembly bonds, joints and alignments
  • d) Battery compartment failing to remain closed (replaceable battery NIS)

11.2 Container Inspection

If the container is completely fractured or has lost contents obviously visible to the user → replace the container to complete the test.

For System Designations C and D: additional samples shall be pre-conditioned to complete the test.

Non-obvious damage → assessed as part of dose accuracy evaluation.

11.3 Dose Accuracy Acceptance Criteria (Final Formulae)

Two-sided (systems A, B2, C, D2):

  • x̄ + k·s ≤ U and x̄ − k·s ≥ L
  • kact = min[(U − x̄)/s, (x̄ − L)/s]

One-sided (B1, D1):

  • x̄ − k·s ≥ L or x̄ + k·s ≤ U
  • kact = (x̄ − L)/s [or (U − x̄)/s for upper limit]

Pass criterion: kact ≥ target k (from Annex B table for selected p and n)

導師解析 Tutorial Commentary

照明條件 215 lux 的意義

ISO 11608-1 規定視覺檢驗必須在 215 ± 20 lux 的環境照明下進行。這個值是典型的辦公室照明水準（一般辦公室 200–500 lux），代表「正常使用環境」。

對 CDMO 的 QC 實驗室意義：

  • 客戶的 DV 測試若在特殊照明下（高或低），結果可能無法代表真實使用
  • CDMO 若幫助客戶進行 QC 視覺檢驗，應確認照明計符合此要求
  • 40–70 cm 的觀察距離是成人正常閱讀距離，需確保 NIS 標示在此距離下清晰可讀

「顯著缺陷」的判定原則

Section 11.1 的缺陷檢查表不僅列出了什麼要看，更重要的是每一項都有限定詞：「影響安全功能（impact safe functioning）」。

這意味著：不是所有外觀缺陷都算作不合格。如果裝置表面有輕微磨損但不影響功能，可能不是「顯著缺陷」。

在 CDMO 協助客戶建立 Release Criteria 時：

  • 視覺缺陷清單應明確區分「外觀缺陷」和「功能性缺陷」
  • 「顯著缺陷」的定義應在 DV 報告的 Acceptance Criteria 中明確說明
  • 可接受/不可接受缺陷的邊界應有照片參考（Limit Sample）

劑量準確性最終判定公式總結

D1 製造商充填（One-sided, lower limit）：
  x̄ - k_act · s ≥ L（最小遞送劑量）
  k_act = (x̄ - L) / s
  通過條件：k_act ≥ k_tar（Annex B, p=0.95, n=20）

判斷示例：
  x̄ = 0.985 mL，s = 0.006 mL，L = 0.970 mL
  k_act = (0.985 - 0.970) / 0.006 = 2.50

  查 Annex B (n=20, p=0.95, one-sided)：
  k_tar ≈ 2.208

  k_act (2.50) > k_tar (2.208) → ✅ PASS
Sections 12–13: 測試報告與標示義務

原文內容 Original Text

12 Test Report (per ISO 13485:2003, Clause 7)

Each test report shall include at minimum:

  1. Reference to this standard (ISO 11608-1)
  2. Identification of physical manufacturer (test initiator)
  3. Identification of NIS tested
  4. Identification of test system used
  5. Identification of test liquid used
  6. Test results, including summary of test conditions
  7. Details of any deviation from ISO 11608-1
  8. Name and address of test facility
  9. Date of test
  10. Dose levels
  11. Number of injectors tested

13.2.2 Marking on the NIS (Device Itself)

Minimum markings on the NIS shall include:

  • Name or trade name of the manufacturer
  • Details to identify the NIS
  • Batch code, lot number or serial number (preceded by appropriate symbol)

13.2.3 Marking on User Packaging

Minimum markings on user packaging:

  • Name and address of manufacturer
  • Sufficient details to identify the NIS
  • Content of user packaging
  • Intended medicinal product information
  • Batch code / lot number / serial number
  • Special storage/handling conditions
  • Expiry date (YYYY-MM format, if required)
  • Single-use indication (if applicable)
  • Specific needles and containers tested for compatibility (if provided separately)

13.3 Instructions for Use (IFU)

IFU must include: all marking information (except lot/expiry), warnings/precautions, step-by-step usage instructions, injection technique guidance, disposal instructions, and storage conditions.

導師解析 Tutorial Commentary

測試報告要求：CDMO 技術轉移的直接影響

Section 12 要求測試報告依據 ISO 13485 第 7 條（設計和開發記錄）保存。對 CDMO 而言：

  • 客戶提供的 DV 報告必須包含所有 a-k 項目；缺少任何一項都是文件不完整的依據
  • 特別注意 e（測試液）和 g（偏差）：若客戶使用代用液，需有充分論證；若有偏差，需有風險評估
  • 「偏差記錄（g 項）」是最常被忽略的：任何對標準測試程序的偏離都必須記錄，即使有合理理由

13.2.3 最後一項的重要性：針頭/容器相容性聲明

用戶包裝標示的最後一項要求（i）：「若分開提供，應標示已測試相容的特定針頭和容器」。

這正是 Introduction 中廢除「Type A 通用相容性」後的直接落地要求：

  • 製造商必須在包裝上明確告知「此裝置已與哪些特定組件測試過」
  • 這為 CDMO 提供了重要的文件審查依據：若客戶變更了針頭或 PFS 供應商，包裝標示是否已更新？
  • 若 CDMO 採購了不同批次或供應商的組件，需評估是否影響這項相容性聲明

IFU（使用說明書）的重要性

IFU 不只是說明書，更是 NIS 設計驗證的一部分：

設計要求 d（劑量指示）、e（預設反饋）、f（注射準備就緒指示）都需要在 IFU 中說明「使用者應如何確認這些指示」。

IEC 62366（可用性工程）要求 IFU 必須通過 Usability Testing（可用性測試）——真實用戶按照 IFU 操作，驗證指示是否清晰、步驟是否可行。這也是為何 IEC 62366 是 ISO 11608-1 的規範性參考。

技術轉移文件審查 Checklist

CDMO 審查客戶提交的 DV 報告時，可使用此快速檢核表：

  • 報告標題包含 ISO 11608-1:2014（或：2012）版本引用
  • 測試液識別（原始藥品或代用液，含論證）
  • 測試系統識別（Gauge R&R ≤ 20% 驗證結果）
  • 所有偏差均有記錄（含偏差理由和風險評估）
  • 劑量準確性結果包含 x̄、s、kact、ktar 和通過/失敗判定
  • 測試矩陣涵蓋所有對應 System Designation 的必做項目
  • 系統相容性聲明（針頭 + 容器 + 裝置組合）已記錄
  • Risk Management File 已更新反映 CDMO 製造環境

Annex A (Informative) — Dose Replicates, Accuracy and Testing Rationale

Annex A provides the statistical rationale for the dose accuracy testing methodology, including:

  • A.1: Dose replicates for variable multi-dose NIS (6 possible replicate sequences: R1–R6)
  • A.2: Accuracy assessment (two-sided tolerance interval formulae for absolute and relative limits)
  • A.3: Testing rationale for each test in Table 3 (why these tests were selected)
  • A.4: Dose delivery efficiency assessment (one-sided, for D1/B1 user-filled)
"For a given test, dose accuracy is evaluated by delivering and measuring Vset in combinations of injection cycles or replicates (random sequences of the three pre-set doses, Vset). A random sequence of three pre-set doses can occur in six possible ways (R1, R2, R3, R4, R5 and R6)."

R1: Vmin, Vmid, Vmax | R2: Vmin, Vmax, Vmid | ... R6: Vmax, Vmid, Vmin

導師解析 Tutorial Commentary

Annex A：為什麼需要 6 個 Replicate 順序？

多劑量 NIS（A/C 系統）的劑量準確性測試中，每支注射器需要按照所有 6 種隨機順序（R1–R6）依次測試三個劑量（Vmin, Vmid, Vmax）。

理由：不同的注射順序會影響容器內的剩餘液量，進而影響下一次注射的驅動條件。通過測試所有組合，可以確認劑量準確性不受注射順序影響。

對 D1/B1 系統（單劑量，全量釋出）：這個問題不存在——每次只注射一次，沒有順序效應。

k 值的統計意義：n 越大 → k 越小

n → ∞ 的 k 值 (p=0.95, one-sided) = 1.645
這是正態分布的 95th percentile（Z分數）

實際測試 n=20：k = 2.208（比1.645大很多）
實際測試 n=60：k = 1.933（更接近理論值）

差異原因：
  n=20 時，用樣本 x̄ 和 s 估計母群體的不確定性更大
  k 值補償這個額外不確定性，確保：
  「在95%信心下，至少95%的母群體在規格內」

  n 小 → 統計不確定性大 → k 大 → 更難通過
  n 大 → 統計不確定性小 → k 小 → 更容易通過

Annex B 是「規範性」附錄！

Annex A 是「informative（參考性）」——用於理解方法論背景。

Annex B 是「normative（規範性）」——必須遵從，具有法律效力。

這意味著 k 值必須從 Annex B 的官方表格中查取。使用其他統計軟體計算的 k 值需要能夠論證與 Annex B 結果一致性，不能任意替換。

實務中，ISO 16269-6（Annex B 引用的統計標準）提供了相同的 k 值，允許使用統計軟體計算，只要方法等效。

比喻：k 值表 vs. 查身高體重比例表

k 值表就像醫學上的「身高對應正常體重範圍表」：告訴你「基於有限的測量樣本，可以對整個人群做什麼程度的聲明」。

樣本人數（n）越多，你的聲明就越精準，需要的「安全係數」（k 值）也越小。

製藥業的統計也遵循同樣邏輯——用有限的批次數據，通過統計容許區間，對整個生產工藝的性能做出有信心的聲明。

綜合練習：完整 DV 場景

  1. 某 adalimumab 生物相似藥的 40 mg/0.8 mL autoinjector（D1 系統）完成了所有 Table 3 必要測試。乾熱/冷儲測試（n=60, p=0.975）的最終結果：x̄ = 0.790 mL, s = 0.004 mL, 下限 L = 0.760 mL（最小遞送劑量）。計算 kact，並與表中 ktar（p=0.975, n=60, one-sided ≈ 2.324）比較，判斷是否通過。
  2. 技術轉移後，CDMO 更換了 PFS 矽化供應商。客戶說「矽化量規格相同，不需要重新 DV 測試」。從 Section 11.1 的 container 標示要求（13.2.3 項 i）和設計要求 u（E&L 風險評估），CDMO 應如何評估和回應？

ISO 11608-1 整體學習總結 Overall Summary

  • 六大系統分類（A/B1/B2/C/D1/D2）決定測試矩陣，D1（製造商充填，整合式，全量釋出）是 CDMO autoinjector 業務最常見類型
  • 統計容許區間（k 值方法）是所有劑量準確性判定的核心；kact ≥ ktar（Annex B）為通過標準；n 越大 → ktar 越小 → 更容易通過
  • Type A 相容性已廢除：製造商必須基於實際測試標示已驗證的組件組合；CDMO 供應商變更需要評估相容性聲明的有效性
  • ISO 14971 + IEC 62366 是強制前提：設計驗證報告必須有對應的 Risk Management File 和 Usability Engineering File
  • 測試報告（ISO 13485 記錄）需包含 11 項最低內容，尤其注意測試液識別和偏差記錄；任何偏差均需有風險評估支撐

III.C Documentation for Parametric Release Process 參數放行製程的文件要求

原文內容 Original Text

The following information specific to the proposed parametric release process should also be included in your submission:

  • A citation to a complete and detailed description of the current relevant terminal sterilization cycle.
  • Identification of the critical process parameters (process/cycle parameters and appropriate load monitors essential for product release) for the product(s) proposed for parametric release, including the minimum and maximum limits for these critical parameters. The critical process parameters should be within the limits that have been validated and approved for sterility assurance of the subject product(s).
  • Acknowledgement that adherence to the critical parameters of the parametric release program will substitute for the performance of a sterility test as the primary release criterion for the product and that sterility test results from the finished product will not be used to overrule any failure to meet the acceptance criteria of the parametric release program. In the event of failure, the specific sterilizer load will be rejected by the quality control unit and will not be released unless there is a provision for reprocessing.
  • Acknowledgement that regardless of the batch release technique used, any specimen tested according to the reference test method for sterility (e.g., compendium or FDA regulations) will meet the criteria for sterility (such as during testing for stability or postmarketing investigations).
  • A description of the sterilization load monitor that indicates the following: (1) the type of monitor being proposed, (2) how the load monitor will be used and analyzed, (3) what functions are being measured by the monitor, and (4) the rationale for the location of the monitor. Additionally, for indirect monitors, we recommend that you include a statement justifying the classification of the indirect indicator that you are using as defined by the American National Standard Institute (ANSI 2008). In certain circumstances a Class 3 indicator may be appropriate; however, a Class 5 indicator is recommended for most situations.
  • Documentation of the control system to verify exposure of the load to the sterilization process.
  • Revision of the certificates of analysis or batch release records for each product subject to parametric release to indicate that parametric release is now the method used to provide assurance of the requirement of sterility. We recommend that you provide a reference to show the link between batch release criteria and the commitments in the application.
"Sterility test results from the finished product will not be used to overrule any failure to meet the acceptance criteria of the parametric release program."

導師解析 Tutorial Commentary

核心概念解析 Key Concepts

七項文件要求的邏輯架構：

  1. 滅菌週期描述引用：不需要在參數放行申請中重複整個驗證報告，但必須提供可追溯的引用連結
  2. CPP 鑑別與限度：每個 CPP 都要有 min/max limits，且必須在已驗證範圍內
  3. 放行原則聲明：這是最關鍵的承諾——CPP 合格即放行，CPP 不合格即拒收，無菌試驗結果不能「推翻」CPP 失敗
  4. 無菌合規聲明：即使採用參數放行，任何依照法定方法進行的無菌試驗仍必須合格（穩定性試驗、上市後調查等）
  5. Load Monitor 描述：四個面向——類型、使用方法、測量功能、位置理由
  6. 控制系統文件：驗證負載已暴露於滅菌製程的控制系統（如 PLC 邏輯、聯鎖裝置等）
  7. CoA/批次紀錄修訂：放行記錄上必須反映參數放行的方法，並連結到申請書中的承諾

比喻說明 Analogy

第三項要求（無菌試驗不能推翻 CPP 失敗）是整個參數放行哲學的基石。想像你通過了一個嚴格的建築安全檢查系統——所有結構計算、材料測試、施工監控都到位。如果最後有一項結構參數不合格（例如鋼筋間距超標），你不能因為「建築看起來沒有裂縫」就說它是安全的。「看起來沒問題」（無菌試驗合格）不能推翻「結構參數不合格」（CPP 失敗）。反之，如果所有結構參數都合格，你不需要在每面牆上鑽洞來證明內部完好。

重點提示 Key Notes

「不能推翻」原則的實務意涵：這意味著如果一批產品的 F₀ 值低於接受標準（例如某探針記錄的 F₀ = 11.8 min，標準為 ≥ 12 min），即使你隨後對該批進行無菌試驗且結果合格，仍然「不能放行」。唯一的出路是 reprocessing（重新滅菌）——前提是你有已核准的 reprocessing 程序且能證明重新滅菌不會影響產品品質 **(PDA TR30, Section 6.3)**。

Load Monitor 的四面向描述：FDA 要求非常具體。例如對 Class 5 化學指示器，你需要說明：(1) 品牌和型號（類型）；(2) 放入滅菌鍋何處、滅菌後何時讀取、判讀標準為何（使用和分析）；(3) 該指示器測量的是「等效於特定 BI 生長/無生長結果的時間-溫度整合效應」（測量功能）；(4) 放在已鑑定的 cold spot 位置的科學理由（位置理由）。

Class 5 vs. Class 3 指示器：Class 5 (Integrating Indicator) 的反應曲線與 BI 高度相關，適用於大多數情境。Class 3 (Specific Test Indicator) 僅在特定條件下反應，FDA 允許「在某些情況下」使用，但明確表示 Class 5 是推薦的標準選擇 **(USP <1222>, Section 4.3)**。

IV. Filing Requirements 送件要求

原文內容 Original Text

To request parametric release in an original application submission, the request should include information specific to parametric release along with sterilization validation information and product release criteria. For changes to an approved application, the request for parametric release should be submitted in a prior approval supplement under 21 CFR 314.70, 21 CFR 601.12 or 21 CFR 514.8(b)(2). The change to parametric release requires FDA approval before its implementation.

If the applicant has current experience using parametric release with a comparable sterilization cycle at the same manufacturing site, and the proposed product's manufacturing process fits into the same validation protocol for parametric release (e.g., container closure system, load patterns, cycle process parameters, and cycle acceptance criteria), then the applicant should meet the filing requirements with a special report for a human drug product, or an annual report for a veterinary drug product or a biologic product. If your product fits into one of these filing categories, contact the review division for your product to verify submission requirements.

導師解析 Tutorial Commentary

核心概念解析 Key Concepts

三種送件途徑：

  1. 原始申請 (Original Application)：在 NDA/ANDA/BLA 初次送件時即納入參數放行。將參數放行資訊與滅菌驗證數據、放行標準一起提交。
  2. Prior Approval Supplement (PAS, 事前核准補件)：已核准產品要從傳統放行轉換到參數放行時的標準途徑。必須「FDA 核准後才能實施」——這是明確的事前核准要求。
  3. Special Report / Annual Report（簡易途徑）：僅適用於已有相同場址、相同滅菌週期的參數放行經驗，且新產品的製程條件（容器、裝載、週期參數、接受標準）完全落入現有已驗證範圍內。

Prior Approval Supplement (事前核准補件)：依 21 CFR 314.70 規定，涉及「影響產品品質的製造方法重大變更」必須以 PAS 提出。從「無菌試驗放行」轉為「參數放行」顯然屬於此類。FDA 審查後核准，申請者才能開始使用參數放行。

比喻說明 Analogy

這三種途徑就像建築使用變更的三種情境：(1) 新建大樓申請時就規劃為餐廳（原始申請）；(2) 原本的辦公大樓要改成餐廳，需要重新申請使用執照（PAS——事前核准）；(3) 同一條商業街上你已經有一家餐廳，隔壁的相同格局店面也要開餐廳，只需要備查報告（Special Report——因為消防系統、通風系統、下水道等都是同一套已驗證的基礎設施）。

重點提示 Key Notes

簡易途徑的條件極為嚴格：想走 Special Report/Annual Report 途徑，必須同時滿足：(1) 同一製造場址；(2) 相同滅菌週期（comparable sterilization cycle）；(3) 容器密封系統、裝載模式、週期參數、接受標準全部在現有驗證方案範圍內。實務上，這意味著同一款滅菌鍋、同類瓶型、同樣裝載配置。

CDMO 策略建議：對於多產品線 CDMO，最有效的策略是：第一個產品以 PAS 建立參數放行基礎（包含寬廣的裝載模式和容器範圍驗證），後續相似產品即可走簡易途徑。這需要在初始驗證方案設計時就預留擴展空間 **(PDA TR30, Section 7.1)**。

務必先聯繫審查部門：FDA 在最後一句明確建議：「contact the review division for your product to verify submission requirements」。這不是形式語言——在決定送件途徑前先與 FDA 溝通，可以避免送件後被要求重新以 PAS 提交的風險。

練習思考 Practice Questions

  1. 一家 CDMO 目前有產品 A 已獲得 FDA 核准的參數放行。現在客戶 B 的產品使用相同滅菌鍋、相同瓶型(50 mL)，但裝載量從每批 5,000 瓶增加到 8,000 瓶（原驗證範圍為 3,000-6,000 瓶）。可以走 Special Report 途徑嗎？為什麼？

導師解析 Tutorial Commentary

重點提示 Key Notes — 參考文獻導讀

以下為 CDMO 實務人員最應深入研讀的核心參考文獻，依實務重要性排序：

1. PDA Technical Report No. 30 (1999)：這是參數放行最完整的技術指引文件。涵蓋 overkill/bioburden-based approach 的選擇、F₀ 計算、load monitor 選型、滅菌週期開發的完整方法論。FDA 本 Guidance 可視為 PDA TR30 的法規配套文件 **(PDA TR30)**。

2. USP <1222> (2009)：美國藥典對參數放行的定義和技術要求。與 FDA Guidance 互為補充——FDA Guidance 告訴你「如何向 FDA 申請」，USP <1222> 告訴你「技術上該怎麼做」**(USP <1222>)**。

3. ICH Q9 (2006)：風險評估的方法學框架。FDA 明確要求風險評估聲明須符合 ICH Q9 原則，因此 FMEA、FTA 等工具的運用不可或缺 **(ICH Q9(R1))**。

4. PIC/S PI 005-3 (2007)：PIC/S 的參數放行專項指引，尤其重要的是 PIC/S Annex 1 (2022 修訂版) Section 8 對最終滅菌的最新要求。CDMO 若有歐洲市場客戶，必須同時滿足 FDA 和 PIC/S 要求 **(PIC/S Annex 1 2022, Section 8.20–8.22)**。

5. ANSI/AAMI/ISO 15882:2008：化學指示器的選型、使用和結果判讀標準。理解 Class 1–6 指示器的分級體系是正確選用 load monitor 的基礎。

6. CPG 460.800 (1987)：FDA 最早接受參數放行的合規政策指引，為本 Guidance 的歷史先驅。瞭解其內容有助於理解 FDA 對參數放行的監管演進。

比喻說明 Analogy

這些參考文獻的關係就像蓋房子所需的各類文件：法規 (21 CFR) 是建築法規——你必須遵守；FDA Guidance 是向市政府提交建築許可申請的指南；PDA TR30 和 USP <1222> 是建築師手冊和工程規範——告訴你如何設計和施工；ICH Q9 是結構安全評估方法；PIC/S 是如果你的房子要在其他國家也合法使用時需要的「國際建築標準互認」。

練習思考 Practice Questions

  3. CDMO 同時服務美國和歐盟市場的客戶。FDA Guidance 和 PIC/S PI 005-3 對參數放行的要求有哪些主要差異？如何設計一套同時滿足兩者的參數放行系統？

核心概念解析 Key Concepts — 跨文獻關聯

本 Guidance 的參考文獻展現了參數放行的「多層次法規架構」：

本節重點回顧 Section Summary

  • 七項文件要求：涵蓋滅菌週期引用、CPP 鑑別與限度、放行原則聲明、無菌合規聲明、load monitor 四面向描述、控制系統文件、CoA/批次紀錄修訂
  • 不可推翻原則：無菌試驗合格結果不能推翻 CPP 不合格的判定——這是參數放行的哲學基石，違反此原則等於否定了參數放行本身的科學基礎
  • Load Monitor 描述四面向：類型 (type)、使用與分析方法 (use and analysis)、測量功能 (functions measured)、位置理由 (rationale for location)；Class 5 為推薦標準
  • 送件途徑選擇：原始申請隨主件提交；已核准產品首次轉換須 Prior Approval Supplement；同場址同週期同類產品可走 Special Report/Annual Report
  • CDMO 策略：第一個產品以 PAS 建立寬廣的參數放行驗證基礎，後續相似產品走簡易途徑；初始驗證方案設計時即預留擴展空間
  • 核心參考文獻：PDA TR30（技術指引）、USP <1222>（藥典標準）、ICH Q9（風險方法）、PIC/S Annex 1（國際調和）構成參數放行的完整知識體系

4.1 Product Description 產品描述

Original Text

The submission file to NBs should include a description of the product as it is intended to be placed on the market (i.e., as a single integral product). The description should contain key information (as in Table 4.1-1🔍) as well as references to more detailed information. The summary may be phrased like this:

Product "Prefilled Syringe XYZ" is intended to be placed on the market in such a way that the medicinal product and the device intended to administer the medicinal product form a single integral product that is intended exclusively for use in the given combination, and is not reusable, as defined by MDR Article 1(9), subparagraph 2.

1 Summary of Product Characteristics

導師解析 Tutorial Commentary

核心概念解析 Key Concepts

Single Integral Product (單一整合產品): 根據 MDR Article 1(9) 第二段，當藥品與用於給藥的裝置形成一個不可分離、僅供該特定組合使用且不可重複使用的整體時，即屬於「單一整合產品」。預充填注射器 (PFS) 就是典型範例 —— 藥液已預先充填於注射器中，兩者構成不可分割的組合。

MAA (Marketing Authorization Application, 上市許可申請): 在歐盟，藥品上市前須取得上市許可。對於 DDC（藥品-裝置組合產品），申請人須在 MAA 中同時提供藥品與裝置部分的資訊。

比喻說明 Analogy

想像一支原子筆：筆芯（藥品）和筆殼（裝置）出廠時已經組裝好，消費者無法也不需要自行更換筆芯。這就像 PFS —— 藥液與注射器是一體的，使用者只需拔開針頭護套即可注射，用完即丟。而 Table 4.1-1🔍 就像是原子筆包裝盒上的產品說明標籤，列出品牌、型號、墨水顏色等關鍵資訊。

重點提示 Key Notes

Table 4.1-1🔍 的內容來自 MAA 文件，提供給 NB 的目的是讓審查員快速了解該整合產品的藥品面概況。注意，NB 審查的重點是裝置部分是否符合 GSPR，但他們仍需要理解藥品的基本資訊，因為藥品與裝置之間存在交互作用（例如藥液與容器的相容性）。

  2. 如果同一支 PFS 要用於兩種不同濃度的藥品，Table 4.1-1🔍 的內容需要如何調整？

導師解析 Tutorial Commentary

核心概念解析 Key Concepts

"Virtual" Device Part（「虛擬」裝置部分）: DDC 在法規上不是醫療器材，而是藥品。但 MDR 要求其裝置部分必須符合 GSPR。因此，業界使用「虛擬裝置」這個概念 —— 假設這個裝置部分是獨立的醫療器材，對它進行分類和評估。對於 staked needle PFS，按 MDR Annex VIII Rule 6 分類為 Class IIa（短暫使用的外科侵入性裝置）。

Staked Needle（固定式針頭）: 針頭在製造過程中即以黏著劑永久固定於注射器筒身，與可拆卸式針頭 (Luer-lock) 不同。這種設計減少了死體積 (dead volume)，降低了針頭脫落的風險。

MDA/MDN 與 MDS/MDT 代碼: 這些是 EMA 用來識別裝置設計用途和技術的編碼系統。MDN 1202 對應注射器類裝置，MDT 2001 對應相關製程技術。

比喻說明 Analogy

「虛擬裝置」的概念就像租屋合約中的「假設獨立產權」評估。雖然這間房間是整棟建築的一部分（就像 PFS 裝置是 DDC 的一部分），但為了評估其安全性，我們假設它是一個獨立單位，檢查它的消防設施、逃生通道是否符合標準。Table 4.2-1🔍 就是這份「假設獨立產權」的身分證明文件。

重點提示 Key Notes

  • Route of Conformity（合規路徑）非常關鍵：DDC 依循 Directive 2001/83/EC（藥品指令），但裝置部分需要 NB 依據 MDR Annex I 的 GSPR 出具意見書。這是藥品法規與器材法規的交會點。
  • Table 4.2-2🔍 定義了裝置的基本屬性，例如「非主動式」(non-active) 表示注射力來自使用者手動推壓，而非電池或彈簧等動力源。
  • Table 4.2-3🔍 強調預期用途與使用者族群的描述必須與 SmPC 保持一致 —— 這是藥品與裝置文件之間的關鍵對齊點。

實務應用 Practical Application

在準備 NB 送件時，藥廠 RA 團隊常見的錯誤是：將裝置部分的描述寫得過於「器材化」，忽略了 DDC 的本質是藥品。例如，不應使用 MDR 器材分類規則來決定合規路徑，而應明確指出這是透過藥品法規（Directive 2001/83/EC）進行評估，NB 僅對裝置部分的 GSPR 出具意見。

練習思考 Practice Questions

  1. 為什麼 TR73-2 強調 DDC 的分類規則「不正式適用」(do not formally apply)？這對 NB 審查有什麼影響？
  2. 如果一款 PFS 設計為可由患者自行在家中使用（home setting），Table 4.2-3🔍 中的「Intended user profile」和「Intended use environment」應如何描述？這對人因工程 (Human Factors) 評估有何影響？
  3. MDA/MDN 和 MDS/MDT 代碼的選擇為何重要？選錯代碼可能導致什麼後果？
4.3 Device Function 裝置功能

Original Text

This section provides a summary of the device function. The description may be phrased like this:

The staked needle PFS consists of a plunger with a rubber stopper that fits tightly in a cylindrical tube (the syringe barrel) containing the drug. To expel the drug, the user pushes the plunger rod until the rubber stopper hits the end of the barrel. Dosing will be fixed, that is, no dose adjustment with respect to bodyweight and/or body surface area is needed. One PFS equals one dose.

導師解析 Tutorial Commentary

核心概念解析 Key Concepts

Fixed Dose（固定劑量）: 「One PFS equals one dose」是 staked needle PFS 的核心特徵之一。每支注射器預充填了精確的單一劑量，使用者不需要也無法調整劑量。這簡化了使用流程，降低了劑量錯誤的風險，但也意味著製造時的充填精度至關重要。

Break-loose Force 與 Glide Force: 雖然本段未展開，但描述中提到的「rubber stopper fits tightly」暗示了兩個重要的力學參數：break-loose force（初始推動力，克服活塞與筒壁的靜摩擦力）和 glide force（持續推動力，維持活塞滑動）。這些力直接影響使用者注射體驗。

比喻說明 Analogy

PFS 的操作原理就像使用一支大型膠水棒：管內的膠水（藥品）被推桿（plunger rod）末端的封塞（rubber stopper）推動，從管口（針頭）擠出。差別在於 PFS 的精度要求極高 —— 每一支都必須確保推出完整且精確的劑量。

重點提示 Key Notes

這段範例措辭雖然簡短，卻包含了 NB 審查時關注的幾個要點：(1) 裝置的基本工作原理、(2) 給藥方式（推壓式）、(3) 劑量策略（固定劑量）。撰寫時應避免過度簡化，但也不需要在此處展開所有技術細節 —— 細節會在後續的 GSPR 合規章節中呈現。

練習思考 Practice Questions

  1. 如果某款 PFS 允許使用者進行劑量調整（例如依體重調整），在 Device Function 描述中需要增加哪些額外資訊？
  2. 「One PFS equals one dose」的設計對於藥品廢棄量（overfill）有何影響？

導師解析 Tutorial Commentary

核心概念解析 Key Concepts

Component（元件）vs Functional Unit（功能單元）: Table 4.4-1🔍 列出了個別元件，而 Table 4.4-2🔍 則將這些元件組合成功能單元。這種分層結構非常重要：

  • 元件層級 (Component level): 單一零件，如活塞 (plunger stopper)、筒身 (barrel)、針頭 (needle)
  • 功能單元層級 (Functional unit level): 由多個元件組成的子系統，如「推桿組件」（活塞+推桿+後擋）
  • 裝置層級 (Device level): 所有功能單元組成完整的注射器

Container-Closure Integrity (CCI, 容器密封完整性): 出現在 plunger stopper 和 needle shield 的描述中，是確保藥品無菌性和穩定性的關鍵指標。

比喻說明 Analogy

這種元件到功能單元的分層結構，就像描述一輛汽車：先列出個別零件（輪胎、煞車片、避震器），再描述功能模組（懸吊系統 = 輪胎 + 避震器；制動系統 = 煞車片 + 碟盤），最後是整車。Table 4.4-1🔍 就是「零件清單」(BOM)，Table 4.4-2🔍 則是「模組組裝圖」。

重點提示 Key Notes

  • Backstop（後擋）標記為「optional component」—— 並非所有 PFS 設計都需要後擋，但如果有，它可以防止推桿被意外拔出。
  • 每個元件都有對應的規格文件參考（Document XXX），這是可追溯性 (traceability) 的要求。NB 審查時可能要求調閱這些文件。
  • 注意 Needle (2B) 的描述提到 "balances injection pain and injection force (flow rate)" —— 針頭規格（gauge、長度、斜面角度）是平衡注射疼痛與藥液流速的關鍵設計要素。

練習思考 Practice Questions

  1. 為什麼 Table 4.4-1🔍 中的 Identifier 使用「1A, 1B, 2A, 2B...」而非簡單的「1, 2, 3...」？這種編碼方式傳達了什麼資訊？
  2. 如果在 PFS 設計中新增一個安全裝置（例如針頭回縮機構），需要在哪些表格中添加資訊？

導師解析 Tutorial Commentary

核心概念解析 Key Concepts

GSPR 10（一般安全與性能要求第10條）: MDR Annex I 中的 GSPR 10 專門針對「化學、物理和生物特性」。當裝置元件含有 CMR 物質、內分泌干擾物 (ED) 或奈米材料時，必須特別標記並提供風險效益分析。

Body Contact 分類: ISO 10993-1 依據接觸類型將材料分類：

  • Surgically invasive（外科侵入性）: 針頭 —— 穿透皮膚進入人體，風險最高
  • Intact skin（完整皮膚接觸）: 推桿、筒身外表面、標籤
  • Indirect body contact（間接人體接觸）: 活塞、針頭護套軟質部分 —— 透過藥液間接接觸人體

Drug Product Contact: 與藥液直接接觸的元件需要進行可萃取物/浸出物 (E&L) 研究。

比喻說明 Analogy

材料接觸分類就像餐廳的食品安全管理：直接接觸食物的器具（相當於 drug product contact = Yes）需要食品級認證；觸碰嘴唇的杯沿（surgically invasive 類比）要求最嚴格的安全標準；而只是握在手中的杯把（intact skin）則要求較低但仍需安全無害。Table 4.5-2🔍 就像一份「特殊成分警示清單」，標記出需要額外關注的材料。

重點提示 Key Notes

  • 針頭 (Needle) 標記含有 CMR 物質 —— 不鏽鋼 (X5CrNi18-9) 中的鎳和鉻屬於 CMR 物質。雖然不鏽鋼已被廣泛使用，但 MDR 要求必須明確揭露並進行風險效益論證。
  • 即時標籤 (Immediate Labeling) 含有奈米材料 —— 這在 MDR 框架下需要額外關注（GSPR 10.6）。可能源自標籤油墨或塗層中的奈米級顏料顆粒。
  • 矽油 (Silicone oil) 作為潤滑劑出現在多個元件中 —— 它影響 glide force、CCI，且可能與某些生物製劑產生交互作用（如蛋白質聚集）。

實務應用 Practical Application

在準備 Table 4.5-1🔍 時，供應商資訊的揭露程度是常見的難題。元件供應商可能不願意完整揭露其配方（屬商業機密）。實務上可採用「第三方機密資料包」(Confidential Data Package) 的方式，讓供應商直接向 NB 提交敏感資訊，而不經過藥廠。這需要事先與供應商和 NB 達成協議。

練習思考 Practice Questions

  1. 為什麼活塞 (plunger stopper) 的 Body Contact 是「No (indirect)」而非「Yes」？如果活塞直接接觸人體皮膚，分類會如何改變？
  2. 針頭的不鏽鋼含有 CMR 物質但仍被允許使用，MDR 如何要求廠商進行風險效益論證？
  3. 如果供應商更換了矽油的品牌或等級，需要更新哪些文件？

導師解析 Tutorial Commentary

核心概念解析 Key Concepts

Tamper-Evident Features（防竄改特徵）: 防竄改設計讓使用者能辨識包裝是否曾被打開或破壞。這是歐盟防偽藥指令 (Falsified Medicines Directive, FMD) 的要求之一，也是 MDR 對包裝完整性的關注點。

GSPR 7 與 GSPR 10.2: 雖然本節指出這些包裝相關的 GSPR 不在 TR73-2 範圍內，但在最終的 MAA 送件中必須處理。GSPR 7 涉及標籤和使用說明書，GSPR 10.2 涉及包裝材料的化學特性。

比喻說明 Analogy

PFS 的包裝層次就像俄羅斯套娃：最內層是泡殼 (blister) 保護注射器，中層是紙盒 (folding box) 提供結構支撐和標示空間，最外層的防竄改標籤 (tamper-evident label) 就像封條，一旦被破壞就無法復原。每一層都有其功能：保護產品、傳達資訊、防止竄改。

重點提示 Key Notes

注意 Table 4.6-1🔍 列出了精確的尺寸資訊 —— 這不僅是 NB 審查需要的，更是確保包裝設計能充分保護 PFS 的基礎。例如，泡殼的形狀和尺寸必須確保 PFS 在運輸過程中不會移動或撞擊，避免針頭護套脫落或玻璃筒身破裂。

Package insert（仿單/使用說明書）的尺寸資訊也很重要 —— 它必須在有限的空間內包含完整的使用指導和安全資訊，同時確保字體大小符合可讀性要求。

  • Description of the manufacturing process, for example, a process flow diagram (main process steps, critical process steps, in-process controls)
  • Processing aids and residuals overview
  • Assembly, labeling, and packaging
  • List manufacturing sites(s), test site(s), site responsible for release, and sterilization sites (if applicable)
  • Sterilization process and validation summary, including decontamination from sterilization residuals
  • Acceptance criteria for batch release (specifications)
  • Change communication policy for specification changes/major process changes

The supplier of the subassembly is responsible for incorporating available information for the respective sourced components, for example, needles, adhesives, lubricants, needle shield, and coatings, in their information package.

4.7.3 Storage and Transport

All device components and packaging materials are stored in compliance with the storage conditions defined by the respective supplier until filling and assembly. All facilities involved in filling and assembly are qualified and perform such functions, for example, as monitoring of temperature, room pressure, and relative humidity. References to storage and transport validation studies performed, as well as simulated and/or real-time studies, should be included. Evidence should be provided to establish that the packaging sufficiently protects the device to enable it to achieve its intended performance.

A summary of the shipping/transit simulation study conducted, and the applicable study reports, should be provided.

For more information, see TR-73, Section 11.9 "Shelf Life and Storage," and Section 12.3.5 "Shipping and Distribution of Semifinished Syringes and PFSs."

導師解析 Tutorial Commentary

核心概念解析 Key Concepts

製造資訊的層次結構: Section 4.7 將製造分為三個層次：

  • 4.7.1 元件與次組件製造 (Component & Subassembly): 由元件供應商執行，包括玻璃管成型、針頭製造、橡膠活塞配方等
  • 4.7.2 DDC 製造 (Filling, Assembly, Packaging): 由藥廠執行，包括無菌充填、推桿組裝、最終包裝
  • 4.7.3 儲運 (Storage & Transport): 涵蓋從元件到最終產品的全鏈條儲存和運輸條件

Change Communication Policy（變更溝通政策）: 這是供應商管理的關鍵要素。元件供應商的製程變更可能影響 DDC 的品質和安全性，因此必須建立明確的變更通知機制。

比喻說明 Analogy

這三個製造層次就像蓋房子：4.7.1 是建材供應商提供合格的磚頭、水泥、鋼筋（各元件製造商的品質保證）；4.7.2 是營造商按照設計圖組裝施工（藥廠的充填和組裝）；4.7.3 則是確保建好的房子在地震、颱風等運輸考驗下仍然穩固（儲運驗證）。NB 需要看到每一層的品質證據。

重點提示 Key Notes

  • 4.7.1 列出了八項元件製造商應提供的最低資訊要求 —— 這是與供應商溝通時的「checklist」。特別注意「change communication policy」，許多 NB 缺陷報告源自供應商未及時通報變更。
  • Table 4.7.1-1🔍 和 Table 4.7.2-1 巧妙地建立了與 TR-73 的交叉參照 —— 這提醒讀者 TR73-2 與 TR-73 是互補文件，TR-73 提供了更詳細的技術指引。
  • 4.7.3 強調了運輸模擬研究 (shipping/transit simulation study) 的重要性。玻璃 PFS 在運輸過程中特別容易受到衝擊和振動的影響，可能導致微裂紋或 CCI 失效。

實務應用 Practical Application

在 CDMO 的業務場景中，4.7 節的準備往往最具挑戰性。作為藥品代工製造商 (CDMO)，你同時扮演兩個角色：(1) 向元件供應商索取製造資訊（作為客戶），(2) 向申請人提供充填組裝的製造資訊（作為供應商）。建議建立標準化的「供應商技術文件包」(Supplier Technical Documentation Package) 模板，確保每次合作都能系統性地收集和提供所需資訊。

本節重點回顧 Section Summary

  • Section 4.0 架構: NB 送件文件依序涵蓋產品描述 (4.1)、裝置識別 (4.2)、裝置功能 (4.3)、元件 (4.4)、材料 (4.5)、包裝 (4.6) 和製造 (4.7) 七大章節，由概況到細節層層遞進。
  • 「虛擬裝置」概念: DDC 不是醫療器材，但其裝置部分需按 MDR 的 GSPR 進行評估。PFS 的「虛擬」分類為 Class IIa（Rule 6），合規路徑依循 Directive 2001/83/EC。
  • 材料與接觸分類: 每個元件需依 ISO 10993-1 標示人體接觸類型和藥品接觸情況，特別關注含有 CMR、ED 物質或奈米材料的元件（GSPR 10）。
  • 製造資訊三層次: 元件製造（供應商負責）、DDC 製造（藥廠負責）、儲運驗證（全鏈條），每層都需提供充分的品質證據和驗證摘要。
  • 與 TR-73 的互補: TR73-2 提供送件框架和範例，TR-73 提供各元件的深度技術指引，兩者應搭配使用。
